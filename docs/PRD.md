# Invoice Web MVP PRD

## 핵심 정보

**목적**: 프리랜서/1인 사업자가 Notion에서 작성한 견적서를 클라이언트가 별도 계정 없이 웹 링크로 즉시 확인하고 PDF로 저장할 수 있게 한다
**사용자**: 클라이언트에게 견적서를 발송하는 프리랜서 및 1인 사업자, 그리고 링크를 받아 견적서를 확인하는 클라이언트

---

## 사용자 여정

```
1. [Notion 데이터베이스]
   ↓ 프리랜서가 견적서 작성 및 발행 처리

2. [견적서 목록 페이지] (프리랜서 전용)
   ↓ 발행된 견적서의 공유 링크 복사

3. 링크를 클라이언트에게 전달 (이메일/메신저)
   ↓

4. [견적서 상세 페이지] (클라이언트 접근)
   ↓ 견적 내용 확인

   [내용 확인 후] → PDF 다운로드 버튼 클릭
                  → [PDF 파일 저장 완료]

5. [완료] → 클라이언트가 PDF 보관 또는 회신
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | Notion 견적서 조회 | Notion API를 통해 특정 견적서 데이터를 가져와 렌더링 | 서비스의 핵심 가치, 데이터 소스 연동 | 견적서 상세 페이지 |
| **F002** | 견적서 목록 조회 | Notion 데이터베이스에서 전체 견적서 목록 조회 | 프리랜서가 발행된 견적서 링크를 관리하는 기반 기능 | 견적서 목록 페이지 |
| **F003** | 공유 링크 생성 | 견적서별 고유 URL 생성 및 클립보드 복사 | 클라이언트에게 견적서를 전달하는 유일한 수단 | 견적서 목록 페이지 |
| **F004** | PDF 다운로드 | 현재 페이지의 견적서를 PDF 파일로 변환하여 다운로드 | 클라이언트의 핵심 니즈, 공식 문서 보관 목적 | 견적서 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F010** | 기본 인증 | 견적서 목록 페이지 접근을 위한 비밀번호 또는 환경변수 기반 간단 인증 | 견적서 목록(관리 화면)을 외부에 노출하지 않기 위한 최소 보호 | 견적서 목록 페이지 |

### 3. MVP 이후 기능 (제외)

- 프로필 상세 관리 (회사 로고, 서명 등 동적 변경)
- 견적서 상태 관리 (승인/거절 등 클라이언트 응답 수집)
- 이메일 직접 발송 기능
- 견적서 편집 기능 (Notion 외부에서)
- 다국어 지원
- 알림 시스템

## 페이지별 상세 기능

### 견적서 목록 페이지

> **구현 기능:** `F002`, `F003`, `F010` | **인증:** 환경변수 기반 간단 인증 필요

| 항목 | 내용 |
|------|------|
| **역할** | 프리랜서 전용 관리 화면. 발행된 견적서 목록을 확인하고 클라이언트에게 전달할 공유 링크를 복사한다 |
| **진입 경로** | 프리랜서가 직접 브라우저에서 접근. 인증 실패 시 접근 차단 |
| **사용자 행동** | Notion DB에서 가져온 견적서 목록을 확인하고, 원하는 견적서의 공유 링크를 복사하여 클라이언트에게 전달한다 |
| **주요 기능** | - Notion 데이터베이스의 견적서 목록 카드/테이블 형태로 표시<br>- 각 견적서의 제목, 클라이언트명, 금액, 작성일, 상태 표시<br>- **공유 링크 복사** 버튼 (클릭 시 클립보드에 URL 복사 및 토스트 알림)<br>- 견적서 상세 페이지로 이동하는 미리보기 링크 |
| **다음 이동** | 공유 링크 복사 후 → 클라이언트에게 외부 전달 / 미리보기 클릭 → 견적서 상세 페이지 |

---

### 견적서 상세 페이지

> **구현 기능:** `F001`, `F004` | **인증:** 없음 (공개 접근, 고유 ID로 접근 제어)

| 항목 | 내용 |
|------|------|
| **역할** | 클라이언트가 링크를 통해 접근하는 공개 견적서 열람 페이지. 견적 내용 확인 및 PDF 저장이 이루어지는 핵심 페이지 |
| **진입 경로** | 프리랜서로부터 공유받은 링크 클릭 (고유 Notion 페이지 ID 기반 URL) |
| **사용자 행동** | 견적서 전체 내용을 읽고, 필요 시 PDF 파일로 다운로드하여 보관한다 |
| **주요 기능** | - Notion API에서 가져온 견적서 데이터를 구조화된 UI로 렌더링 (발행사 정보, 클라이언트 정보, 항목별 금액 테이블, 합계, 메모)<br>- 인쇄/PDF에 최적화된 레이아웃 (여백, 폰트, 컬러 시스템)<br>- **PDF 다운로드** 버튼 (브라우저 Print API 또는 html2pdf 라이브러리 활용)<br>- 존재하지 않는 ID 접근 시 404 안내 메시지 표시 |
| **다음 이동** | PDF 다운로드 완료 → 동일 페이지 유지 / 잘못된 접근 → 오류 안내 페이지 |

### 오류 안내 페이지

> **구현 기능:** 없음 (라우팅 처리) | **인증:** 없음

| 항목 | 내용 |
|------|------|
| **역할** | 존재하지 않는 견적서 ID 또는 잘못된 URL 접근 시 사용자에게 안내를 제공한다 |
| **진입 경로** | 유효하지 않은 견적서 ID로 견적서 상세 페이지 접근 시 자동 리디렉션 |
| **사용자 행동** | 오류 메시지를 확인하고 홈 또는 이전 페이지로 이동한다 |

| **다음 이동** | 홈 링크 클릭 → 견적서 목록 페이지 (인증 필요) |

## 데이터 모델

Notion을 유일한 데이터 소스로 사용하며, 별도 데이터베이스는 MVP에서 구축하지 않는다. Notion API 응답을 아래 구조로 매핑하여 사용한다.

### Invoice (견적서)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | Notion 페이지 고유 ID | string (UUID) |
| title | 견적서 제목 | string |
| clientName | 클라이언트명 | string |
| issuerName | 발행사(프리랜서)명 | string |
| issueDate | 발행일 | string (date) |
| dueDate | 지급 기한 | string (date) |
| status | 견적서 상태 (초안/발행) | string |
| totalAmount | 합계 금액 | number |
| items | 견적 항목 목록 | InvoiceItem[] |
| notes | 추가 메모 | string |

### InvoiceItem (견적 항목)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 항목 고유 ID | string |
| description | 항목 설명 | string |
| quantity | 수량 | number |
| unitPrice | 단가 | number |
| amount | 항목 금액 (수량 x 단가) | number |

## 기술 스택 (최신 버전)

### 프론트엔드 프레임워크

- **Next.js 15** (App Router) - React 풀스택 프레임워크, SSR/SSG 활용
- **TypeScript 5.6+** - 타입 안전성 보장
- **React 19** - UI 라이브러리 (최신 동시성 기능)

### 스타일링 & UI

- **TailwindCSS v4** (설정 파일 없는 새로운 엔진, `globals.css` 기반) - 유틸리티 CSS
- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리 (New York 스타일)
- **Lucide React** - 아이콘 라이브러리

### 폼 & 검증

- **Zod** - 환경변수 및 API 응답 스키마 검증

### 데이터 연동

- **@notionhq/client** - Notion 공식 API 클라이언트 (서버 사이드 전용)

### PDF 생성

- **브라우저 Print API (`window.print()`)** - 기본 PDF 저장 방식 (의존성 없음)
- 또는 **html2pdf.js** - 클라이언트 사이드 HTML-to-PDF 변환 (복잡한 레이아웃 필요 시)

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼, 환경변수 관리

### 패키지 관리

- **npm** - 의존성 관리
